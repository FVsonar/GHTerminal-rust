## 1. 概述

1. 数据通信通过 Q900 内置声卡传输，可以通过声卡传输、读写数据。电台设置为 USB 模式的时候传输调制数据，电台设置为 SDR 模式的时候传输 IQ 数据。
2. 控制协议数据可以通过蓝牙 SPP、BLE、RS232、USB 接口进行控制，协议遵循串口标准。

### BLE 配置

#### V1.0 硬件

| 项目 | UUID |
|------|------|
| 服务 UUID | 0000FFF0-0000-1000-8000-00805F9B34FB |
| 写特征 | 0000FFF2-0000-1000-8000-00805F9B34FB |
| 通知特征 | 0000FFF1-0000-1000-8000-00805F9B34FB |

#### V2.0 硬件

|------|------|------|

| 特征 UUID | FFE2 | 用于音频蓝牙或 SD 卡音乐播放控制，属性 Write |

## 2. 协议格式

| 包头 | 包头 | 包头 | 包头 | 包长 | 命令类型 | DATA | CRC 高 | CRC 低 |
|------|------|------|------|------|----------|------|--------|--------|
| 0xA5 | 0xA5 | 0xA5 | 0xA5 | 1 字节 | 1 字节 | 可变 | 1 字节 | 1 字节 |

- **包头**：分别用四个 0xA5 作为包头
- **包长**：一个字节（BYTE），表示从包长下一个字节开始到包尾的字节长度
- **命令类型**：见协议内容
- **DATA**：见协议内容
- **校验**：采用 CRC 校验方式，从包长到 CRC 高字节前一个字节，算法见附录一

### 3.4 频谱数据命令

#### APP 发送

| 包头 | 包头 | 包头 | 包头 | 包长 | 命令码 | CRC校验码 | CRC校验码 |
| ---- | ---- | ---- | ---- | ---- | ---- | ----- | ----- |
| 0xA5 | 0xA5 | 0xA5 | 0xA5 | 包长 | 0x39 | CRC高 | CRC低 |

#### 电台回复

| 包头 | 包头 | 包头 | 包头 | 包长 | 命令码 | 频谱数据 | CRC校验码 | CRC校验码 |
| ---- | ---- | ---- | ---- | ---- | ---- | -------- | -------- | -------- |
| 0xA5 | 0xA5 | 0xA5 | 0xA5 | 包长 | 0x39 | 频谱数据 | CRC高 | CRC低 |

> **说明：**
> 回复的包长目前固定为 0（APP 发送时为正确计算的实际包长），实际频谱数据为 512 字节。
> CRC 校验目前是占位，无实际作用。

* **V1.0 硬件（已弃用）**
  频谱数据包长 256 个字节，无包头无校验。

* **V2.0 硬件（已弃用）**
  频谱数据包长 80 个字节，无包头无校验。

* **最新硬件（正在使用）**
  频谱数据包长 512 个字节，有包头、有包长（无作用，固定为 0，占位）、有命令码、有 CRC 校验（无作用，占位）。整个数据包加上包头、包长、命令码、CRC 校验共 520 字节。

* **频谱图**：大小代表 Y 轴高度，位置代表 X 位置绘图。

* **瀑布图**：大小代表颜色（蓝色 + 当前值），位置代表 X 位置绘图。

### 3.5 状态同步命令

#### APP 发送

| 0xA5 | 0xA5 | 0xA5 | 0xA5 | 包长 | 0x0B | CRC 高 | CRC 低 |
|------|------|------|------|------|------|--------|--------|

#### 电台回复

| 0xA5 | 0xA5 | 0xA5 | 0xA5 | 0xA5 | 包长 | 0x0B | 收发状态 | VFOA 模式 | VFOB 模式 | VFOA 频率 | VFOB 频率 | A/B | NR/NB |
|------|------|------|------|------|------|------|----------|-----------|-----------|-----------|-----------|-----|-------|

| RIT | XIT | 滤波器带宽 | 频谱带宽 | 电压 | UTC 时间 (时) | UTC 时间 (分) | UTC 时间 (秒) | 状态栏 | S 表/PO 表值 | SWR/AUD/ALC | CRC 高 | CRC 低 |
|-----|-----|------------|----------|------|---------------|---------------|---------------|--------|--------------|-------------|--------|--------|

**字段说明：**

| 字段 | 长度 | 说明 |
|------|------|------|
| 收发状态 | 1 字节 | 0：接收状态，1：发射状态 |
| VFOA 模式 | 1 字节 | 0:USB, 1:LSB, 2:CWR, 3:CWL, 4:AM, 5:WFM, 6:NFM, 7:DIGI, 8:PKT |
| VFOB 模式 | 1 字节 | 同上 |
| VFOA 频率 | 4 字节 | 最大值十进制 2000000000 |
| VFOB 频率 | 4 字节 | 最大值十进制 2000000000 |
| A/B | 1 字节 | 0:A 频，1:B 频 |
| NR/NB | 1 字节 | 0:NR/NB 关，1:NR 开，2:NB 开 |
| RIT | 1 字节 | 0~120 |
| XIT | 1 字节 | 0~120 |
| 滤波器带宽 | 1 字节 | 0~50（序号对应滤波器见附表） |
| 频谱带宽 | 1 字节 | 0:48K, 1:24K, 2:12K, 3:6K, 4:3K, 5:1.5K |
| 电压 | 1 字节 | 十进制值/10 |
| UTC 时间 | 3 字节 | 时:0~24, 分:0~60, 秒:0~60 |
| 状态栏 | 1 字节 | Bit0:蓝牙，Bit1:GPS，Bit2:LORA，Bit3:电子罗盘，Bit4:天调，Bit5:功率 |
| S 表/PO 表值 | 1 字节 | 接收时为 S 表 (BIT7=0)，发射为 PO 表 (BIT7=1)，0~34 |
| SWR/AUD/ALC | 1 字节 | BIT7,BIT6: 00=SWR, 01=ALC, 10=ADU，0~34 |

**状态栏位定义：**

| Bit | 1 | 0 |
|-----|---|---|
| Bit0 | 蓝牙连接成功 | 蓝牙断开 |
| Bit1 | GPS 模块在线 | GPS 模块断线 |
| Bit2 | LORA 模块在线 | LORA 模块断线 |
| Bit3 | 电子罗盘模块在线 | 电子罗盘模块断线 |
| Bit4 | 天调开启 | 天调关闭 |
| Bit5 | 高功率 | 低功率 |

## 附录 1：CRC 校验算法

结果验证网址：http://www.ip33.com/crc.html

**校验模式**: CRC16/CCITT-FALSE

```c
/****************************************************
** 函数名称: CRC16Check  
** 输    入: buf 要校验的数据;
**             len 要校验的数据的长度
** 输    出: 校验值
** 功能描述: CRC16 循环冗余校验 
** 备注：校验模式为 CRC16/CCITT-FALSE，注意变量类型
*****************************************************/
unsigned int CRC16Check(unsigned char *buf, unsigned char len)
{
    unsigned char  i, j;
    unsigned int uncrcReg = 0xFFFF;
    unsigned int uncur;
    for (i = 0; i < len; i++)
    {
        uncur = buf[i] << 8;
        for (j = 0; j < 8; j++)
        {
            if ((int)(uncrcReg ^ uncur) < 0)
            {
                 uncrcReg = (uncrcReg << 1) ^ 0x1021;
            }
            else
            {
                  uncrcReg <<= 1;
            }               
            uncur <<= 1;           
        }
    }
    return uncrcReg;
}
```
